|CONFIG_WATCH_QUEUE|它用于启用/禁用对Watch Queue的支持。如果该选项被启用，则内核将支持使用Watch Queue来管理事件队列，以便在多个线程之间共享事件 。|我也觉得要开|
|CONFIG_SCHED_CORE|用于启用/禁用对核心调度的支持。如果该选项被启用，则内核将支持使用核心调度算法来管理CPU的负载和资源分配 。|我觉得要开
|2024-7-11
|CONFIG_PAHOLE_VERSION|某些相关配置要用，编译 linux 内核时若选上了 config_debug_info_btf，就需要安装 pahole 用于生成.tmp_vmlinux.btf|没有安装dwarves版本配置项会变成0
|CONFIG_USELIB|启用老旧的 uselib() 系统接口支持|
|CONFIG_NO_HZ_FULL|它使内核能够避免向具有单个可运行任务的 CPU 发送调度时钟中断。这些 CPU 被称为“自适应滴答 CPU”。|启用后可以提高实时性能
|CONFIG_BPF_LSM|它表示自愿内核抢占（Voluntary Kernel Preemption）。这种模式通过增加抢占点的方式来减小延迟。在内核中选择性地增加了一些抢占点（使用了might_sleep()宏），目的是减小最大调度延迟，并对交互事件提供更快的响应，代价是会稍微降低吞吐量。|
|CONFIG_VIRT_CPU_ACCOUNTING|CONFIG_VIRT_CPU_ACCOUNTING是 Linux 内核中的一个配置选项，用于启用虚拟 CPU 时间和状态的统计功能。它涉及对 CPU 和进程的时间及状态进行统计和记录。具体来说，启用该选项后，内核可以更详细地跟踪和记录每个任务（进程）在 CPU 上的执行时间、各种状态的时间分布等信息。|
CONFIG_RCU_EXPERT|启用该选项后，才可以设置一些与 RCU（Read-Copy Update，读-复制-更新）相关的更高级或专业的参数和特性。RCU 是一种同步机制，常用于替代读写锁。它的特点是更新操作（写端）不会阻塞读取操作（读端），从而可以提高读操作的性能和可扩展性。|感觉是对读写性能提高有帮助
CONFIG_PRINTK_INDEX|一种无锁同步机制。在某些情况下，为了进一步优化系统性能或满足特定需求，可能需要对 RCU 的行为进行调整。这样做会给运行这些 rcuo（gps）内核线程和处理锁定竞争的 CPU 带来特定的开销。然而，这种方式可以在一些场景中带来好处，例如对于具有严格实时响应约束的应用程序，它允许通过调度时钟中断的最大持续时间来改善其最坏情况响应时间，对于计算密集型的短迭代工作负载也可能很重要。|感觉还是提高实时响应的
CONFIG_PM_GENERIC_DOMAINS| Linux 内核中与电源管理相关的一个配置选项。当启用此选项时，它允许定义电源域，即可以共享同一电源状态的一组设备。这可以实现更有效的电源管理，通过同时将多个设备转换到低功耗状态，而不是单独管理每个设备的电源状态。|得开
CONFIG_MODULE_SIG_ALL|驱动模块签名|不关容易编译不过去
CONFIG_MODULE_FORCE_LOAD|模块强制加载|没有的设备还是不加载了，加速开机|
CONFIG_HAVE_FAST_GUP|优化gpu访问内存特性|要开
